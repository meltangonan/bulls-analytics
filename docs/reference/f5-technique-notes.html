F5 Technique Notes

Craft patterns distilled from Owen Phillips' newsletter The F5 (thef5.substack.com), in our own words, each mapped to where it lives in this repo. The newsletter ended August 2025; its ~75 tutorials remain at the tutorial index (paid archive — these notes exist so the knowledge survives without it).

House rule: we take his structure (layout craft, labels, thresholds), not his look. Bulls graphics keep Playfair Display / DM Sans, Bulls red, and the 4:5 portrait format. No article text is reproduced here — notes and links only.

Craft patterns

1. Headshots and logos as data labels Built

Player headshots and team logos sit directly on the graphic as row labels, replacing text-only identification. This is the single biggest legibility upgrade for social-feed graphics — a face reads faster than a name.

When
Any leaderboard, board, or table row keyed to a player or team.
Source
How To Create Instagram Eye Candy (his ggpath usage)
In repo
bulls.graphics.craft.headshot_label (takes an image path; placeholder circle when missing); ids resolve via bulls.data.fetch.get_player_headshot with local caching.

2. Gradient-scaled stat bars and columns Built

The bar or table cell encoding a stat is colored by its magnitude on a light→dark ramp, so the ranking reads twice: by position and by color depth. He anchors the ramp to one hue rather than a rainbow.

When
Leaderboard value bars; the "verdict" column of a stats table (e.g., net rating).
Source
Instagram Eye Candy (scale_fill_gradient); Great Tables In Python (data_color)
In repo
bulls.graphics.craft.gradient_bar (ramp anchored on Bulls red); color-scaled columns in scripts/prototypes/f5_lineup_table.py.

3. Stacked primary + context labels Built

Two-line labels — bold player name over a muted context line (team, minutes, role) — pack identification and qualification into one visual unit instead of extra columns.

When
Any row where a name needs context without widening the layout.
Source
How To Make Beautiful Big Boards (gt_merge_stack)
In repo
bulls.graphics.craft.stacked_label — includes the house 16-character truncation rule (long names become "F. Lastname") so every builder handles names identically.

4. Thresholds and coverage printed on the graphic Built

Qualification rules live in the subtitle or footer of the image itself ("min. 3 stat boards and 3 scout boards", "minimum 50 summer-league minutes"), never only in a caption. This is our fairness-guardrail #4 executed as a reusable element.

When
Every graphic, by default.
Source
Big Boards; Small multiples in R and Python
In repo
bulls.graphics.craft.threshold_footer (threshold + coverage window + source line).

5. Outlined value text on busy backgrounds Built

Big stat numbers get a thin contrasting outline so they stay legible on top of gradient bars or photos. His R tool is shadowtext; the matplotlib equivalent is a stroke path-effect on the text object.

When
Value labels drawn over colored bars or images.
Source
Instagram Eye Candy
In repo
Applied inline in scripts/prototypes/f5_leaderboard.py via matplotlib.patheffects.

6. Polished tables as post format Built

A well-set table — logos in cells, stacked labels, one color-scaled verdict column, row striping, tight padding, "UR"-style explicit markers for missing values — is a first-class Instagram format, not a fallback. His stack is {gt}/great_tables; ours is plottable (matplotlib-native, so it reuses our fonts and save pipeline without a browser dependency).

When
Multi-column evidence: lineups, combo stats, side-by-side player comparisons.
Source
Great Tables In Python; More Great Tables in R and Python (headshots as column headers); Big Boards (spanners, borders, color domains)
In repo
scripts/prototypes/f5_lineup_table.py.

7. Small multiples with shared scales Parked

One mini-chart per player/team in a grid, ordered by a meaningful stat (he orders by correlation strength or total rank), with axis limits forced equal across facets so panels compare honestly. His trick: a dummy dataset carrying each facet's min/max.

When
Season-shape stories across the roster; distribution comparisons.
Source
Small multiples in R and Python; NBA Jam Style Small Multiple
In repo
Not yet built — catalog card "Season Trend Small Multiples" (Parked). Matplotlib path: plt.subplots grid + shared limits.

8. Percentile bars / stat cards Parked

Each player gets a mini-card: headshot, name plate, and 3–5 horizontal percentile bars (0–100 vs league) in rounded-bar style with a faint full-scale track behind each bar. Reads like a video-game rating card — extremely feed-friendly.

When
Player profiles; rotation overviews; "who does what" explainers.
Source
NBA Jam Style Small Multiple (ggchicklet rounded bars + percentile ranks)
In repo
Not yet built — catalog card "Bulls Percentile Cards" (Parked). Matplotlib path: FancyBboxPatch for rounded bars.

9. High-resolution 4:5 export Built

He exports at 4×5 inches, 600 DPI — the same 4:5 shape as our 1080×1350, just denser. Instagram recompresses everything to ~1080 wide, and text drawn at higher resolution survives that compression visibly crisper.

When
All new prototypes.
Source
Instagram Eye Candy
In repo
save_feed_post(fig, path, dpi=300) → 2160×2700. Existing builders stay at 150 DPI until restyled.

10. The engagement-format anatomy — used knowingly Built

His most-shared format: top-10 on one stat, one big number per row, faces + logos, zero interpretation required in the first second. He is candid that this is bait — a stat-as-factoid, not analysis. We use the format as a hook but keep our guardrails on it: visible threshold, coverage window, source line, and an insight-first caption.

When
Reach posts; niche-stat leaderboards; season milestones.
Source
How To Create Instagram Eye Candy
In repo
scripts/prototypes/f5_leaderboard.py.

Data sources he leans on

pbpstats.com free JSON API — niche play-by-play-derived stats (the Z-Bounds leaderboard came from api.pbpstats.com/get-totals/nba). Our fallback when nba_api lacks a stat.

His GitHub data repo — github.com/Henryjean/data: square team logos, referee data, tutorial datasets. Third-party hosted; cache locally anything we adopt (as we already do for headshots).

Full tutorial index — thef5.substack.com/p/r-tutorials: ~75 tutorials (2020–2025) including formats not yet mined here: diamond plots, beeswarms, availability charts, comet plots, matchup tables, shot-clock data, DuckDB play-by-play work.
